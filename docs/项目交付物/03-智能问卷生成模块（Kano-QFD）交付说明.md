# 智能问卷生成模块（Kano-QFD）交付说明

## 1. 模块定位

本模块负责根据项目上下文、需求澄清结果和已选算法组合，自动生成可编辑的结构化问卷，并将其作为后续发布、采集和分析的正式输入。

本模块面向的核心业务链路为：

- 需求算法：Kano
- 决策分析链路：QFD 或 AHP+QFD

需要强调的是，当前系统遵循“问卷由需求算法定义，决策算法在后续阶段消费结果”的设计原则，问卷生成阶段不会直接做 QFD 计算。

## 2. 当前已实现能力

### 2.1 功能能力
- 基于管线上下文自动生成问卷。
- 读取算法配置并调用相应的需求分析能力生成问卷。
- 对返回问卷执行题型约束校验。
- 将问卷内容持久化。
- 支持在草稿阶段编辑问卷并自动生成新版本。
- 支持后续发布为公开问卷链接。

### 2.2 主要入口
- 页面入口：`/pipelines/[id]`

## 3. 处理流程

1. 用户在管线中完成需求澄清并确定算法组合。
2. 系统检查当前管线状态是否允许生成问卷。
3. 系统从算法组合中解析需求算法。
4. 系统调用需求分析能力生成问卷草稿。
5. 系统校验问卷结构与题型约束。
6. 生成成功后保存问卷 Artifact，并推进研究状态。
7. 用户可在前端编辑器继续调整问卷内容。

## 4. 当前适配情况

### 5.1 已适配题型
- Kano 题
- 单选题
- 多选题
- 量表题
- 文字题

### 5.2 需求算法 Kano 与 决策算法 QFD 的关系
- Kano 负责定义采集哪些用户需求和感知信息。
- QFD 在后续工作流中消费采集结果与计算结果。

## 5. 页面与交互表现

在管线详情页中，问卷以右侧面板形式展示，用户可完成：

- 查看问卷标题、说明、Section 和题目。
- 调整题目文本与顺序。
- 保存修改并形成新版本。
- 发布问卷形成公开填写链接。
