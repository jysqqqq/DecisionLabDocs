# 客户反馈数据收集模块（AI）交付说明

## 1. 模块定位

本模块负责承接已发布问卷的外部填写、匿名提交、响应持久化、采集进度监控和题目分布统计，是连接问卷生成与决策分析之间的中间环节。

本模块的智能化价值主要体现在以下方面：

- 采集前：由智能问卷生成模块产出采集载体。
- 采集后：为报告生成和决策分析提供结构化输入。

因此，该模块能够作为智能决策链路中的客户反馈数据收集模块对外交付。

## 2. 当前已实现能力

### 2.1 公开采集能力
- 支持将问卷发布为公开链接。
- 外部填写者无需登录即可访问。
- 支持是否允许重复提交配置。
- 支持关闭采集后链接失效。

### 2.2 采集监控能力
- 提供跨项目采集监控中心。
- 按问卷版本聚合展示采集情况。
- 展示总样本量、目标样本量、渠道数量、最近更新时间。
- 支持按项目、状态、关键词筛选。

### 2.3 数据分析能力
- 统计单选、多选、量表、文本题、Kano 双向题分布。
- 提供文本题关键词提取和词云展示。
- 展示渠道级贡献分布。
- 支持版本历史查看和详情页分析。

## 3. 主要入口

- 列表页：`/data-acquisition`
- 详情页：`/data-acquisition/[artifactId]/[version]`
- 公共问卷页：`/q/[slug]`
## 4. 业务流程

1. 管线问卷草稿发布为公开链接。
2. 外部用户通过 `/q/[slug]` 访问并填写问卷。
3. 系统校验问卷是否仍处于有效采集状态。
4. 提交结果以结构化 `answers` 形式落库。
5. 采集中心按问卷版本聚合统计提交数据。
6. 用户可在后台查看题目分布、文本词云和渠道贡献。
7. 当样本满足要求或人工结束时，系统关闭采集并推进管线状态。

## 6. 模块特点

- 支持公开采集与匿名填写，适合教学研究和用户调研场景。
- 支持版本化采集统计，便于不同问卷版本独立管理。
- 支持与后续分析、报告模块无缝衔接，形成完整数据链路。

## 7. 当前边界

- 不在采集阶段直接给出业务结论。
- 不在采集详情页直接进行问卷结构编辑。
- 不在当前模块内完成最终 Kano 分类结论或 QFD 计算。
