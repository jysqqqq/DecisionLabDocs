## 3. 当前实际架构特征

### 3.1 全栈一体架构
项目采用 Next.js App Router 统一承载页面、API Route Handlers 和部分服务逻辑，减少前后端拆分带来的集成成本。

### 3.2 数据结构化存储
系统中的问卷、采集结果、计算结果、报告内容大量使用 JSON 存储，既能保证结构化渲染，又利于算法工具扩展。

### 3.3 可配置智能能力
系统采用统一的智能能力接入机制，支持：

- 统一配置大模型服务与模型参数。
- 统一管理需求算法和决策算法的接入信息、版本与协议。
- 根据业务场景调用相应的智能服务与算法能力。

## 5. 数据模型清单

当前数据库核心模型包括：

- `User`、`Account`、`Session`：用户与认证。
- `Project`、`ProjectSettings`、`ProjectReport`：项目与项目级报告。
- `Pipeline`、`PipelineMessage`、`PipelineContext`、`PipelineArtifact`、`PipelineEvent`：决策管线主链路。
- `QuestionnairePublish`、`QuestionnaireResponse`：问卷发布与响应。
- `AlgorithmTool`：算法工具注册表。
- `SystemLLMConfig`：系统级 LLM 配置。
- `DecisionWorkflow`、`DecisionStep`：QFD/决策工作流。
