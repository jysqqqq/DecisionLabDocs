# 可视化报告生成模块（AI）交付说明

## 1. 模块定位

本模块负责将问卷采集结果、计算结果以及 QFD 工作流中间产物整合为面向人阅读的分析报告，并提供结构化展示、章节化阅读、导出和项目总报能力。

## 2. 当前已实现能力

### 2.1 管线级报告
- 读取 calc 结果并生成结构化报告 Artifact。
- 根据 QFD 轮次和控制计划映射构建报告快照。
- 生成固定章节的研究/产品型分析报告。
- 支持多种 Section 类型渲染，包括文本、排序、评分、矩阵、洞察。
- 支持在管线详情页查看和下载报告。

### 2.2 项目级总报
- 聚合多个已完成管线报告。
- 借助 LLM 生成项目综合总结。
- 自动追加引用标记和来源映射。
- 支持项目总报查看与导出。

### 2.3 可视化与结构化展示
- 章节分组展示。
- 排名、评分、矩阵和洞察卡片渲染。
- 支持面向服务业和制造业两类 QFD 章节模板差异化输出。

## 3. 主要入口

- 管线报告接口：`GET/POST /api/pipelines/[pipelineId]/report`
- 项目总报接口：`GET/POST /api/projects/[id]/summary-report`
- 管线报告展示：管线详情页 Report 面板
- 项目总报展示：项目详情页报告抽屉

## 4. 报告生成流程

1. 系统读取最新的 calc Artifact。
2. 系统提取工作流轮次、关系矩阵、优先级和制造映射信息。
3. 系统结合项目、管线和行业上下文构建报告快照。
4. 通过 LLM 生成章节摘要、发现与启示。
5. 将内容组织为报告结构。
6. 将内容持久化。
7. 前端按 Section 类型进行可视化渲染和导出。

## 5. 当前支持的报告内容

### 6.1 管线级报告内容
- 项目背景与研究目标
- 问卷与需求分析摘要
- 多轮 QFD 章节分析
- 制造落地或服务流程展开结果
- 总结与建议

### 6.2 项目级总报内容
- 项目背景与目标
- 方法综述
- 各管线核心发现
- 跨管线对比
- 综合洞察与建议
- 参考来源映射
