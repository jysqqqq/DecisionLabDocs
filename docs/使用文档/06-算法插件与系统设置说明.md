# 算法插件与系统设置说明

## 1. 适用对象

- 管理员

## 2. 模块目标

本模块用于管理系统中的算法工具和模型服务配置，确保决策管线能够正常完成问卷生成、智能分析和报告生成。

## 3. 算法插件管理

### 3.1 进入方式

在左侧导航点击“算法引擎 (Algorithms)”。

### 3.2 页面功能

该页面支持：

- 搜索算法
- 按类型筛选
- 按启用状态筛选
- 按来源筛选
- 新增算法工具
- 编辑算法工具
- 启用/禁用算法
- 删除算法

## 4. 新增算法工具

### 4.1 适用场景

- 接入新的需求分析算法
- 接入新的决策分析算法

### 4.2 操作步骤

1. 点击“新增算法”。
2. 填写算法基本信息。
3. 选择算法类型：
   - 需求算法
   - 决策算法
4. 填写版本、接口地址、认证方式等。
5. 保存提交。

### 4.3 使用建议

- 命名应清晰反映算法用途。
- 版本号建议采用统一规范。
- 接口地址和认证信息应由管理员统一维护。

## 5. 编辑算法工具

管理员可在算法列表中对算法进行编辑，常见修改内容包括：

- 算法名称
- 说明文字
- 接口地址
- 超时时间
- 适用场景配置

## 6. 启用与禁用

### 6.1 启用算法
当某算法通过测试并准备投入使用时，可设置为启用状态。

### 6.2 禁用算法
当某算法暂不稳定、接口维护中或不希望在系统中继续使用时，可暂时禁用。

## 7. 删除算法

删除操作适用于测试性或废弃的算法工具。正式使用中的算法建议优先禁用而非直接删除。

## 8. 系统设置

### 8.1 进入方式
在左侧导航点击“系统设置 (Settings)”。

### 8.2 可设置内容

- 模型服务提供方
- 模型名称
- API Key
- 启用状态

### 8.3 常用操作流程

1. 选择模型服务提供方。
2. 输入或更新 API Key。
3. 点击刷新模型列表。
4. 选择需要使用的模型。
5. 测试连接。
6. 保存配置。

## 9. 系统信息查看

在系统设置页可查看基础系统信息，例如：

- 系统版本
- 运行环境
- 数据库连接状态
- 运行时长
- 内存占用

## 10. 建议维护流程

建议管理员定期执行以下维护动作：

1. 检查模型服务是否可用
2. 检查关键算法是否处于启用状态
3. 清理不再使用的测试算法
4. 更新必要的接口配置与密钥

## 11. 常见问题

### 11.1 模型列表刷新失败
请检查网络连接、API Key 是否有效，以及外部模型服务是否可访问。

### 11.2 算法创建后无法使用
请检查：

- 是否已启用
- 接口地址是否正确
- 认证信息是否完整
- 算法类型是否选择正确

### 11.3 什么时候需要进入系统设置
通常在系统首次部署、模型服务变更或接口密钥更新时需要进入系统设置。
